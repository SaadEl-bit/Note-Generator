Python vs C / Java — What Changes

Since you know C and Java, the transition is mostly about unlearning strict rules. Python is dynamically typed, interpreted, and reads almost like pseudocode. Here's a direct translation of key concepts:

Concept C / Java Python
Variable int x = 5; x = 5
String String s = "hi"; s = "hi"
Array / List int[] arr = {1,2,3}; arr = [1, 2, 3]
Print System.out.println("hi"); print("hi")
For loop for(int i=0; i<5; i++) for i in range(5):
Function public int add(int a, int b) def add(a, b):
Class public class Agent {} class Agent:
Null null / NULL None
Boolean true / false True / False
Semicolons Required ; Not needed
Blocks { curly braces } Indentation (4 spaces)
Import import java.util.*; import json
⚠️
Indentation is mandatory and meaningful in Python. Unlike C/Java where braces define blocks, Python uses 4 spaces. A wrong indent = a crash. Always configure your editor to use spaces, not tabs.

Functions

PYTHON — Functions & Default Arguments def keyword 
# Basic function
def clean_note(raw_text: str) -> str:
    """Remove extra whitespace and empty lines."""
    lines = raw_text.strip().split("\n")
    cleaned = [line.strip() for line in lines if line.strip()]
    return "\n".join(cleaned)

# Default arguments (not available in C, like in Java)
def build_prompt(note: str, style: str = "markdown", max_words: int = 500) -> str:
    return f"""
Convert this note to {style} format in max {max_words} words:

{note}
    """

# Call with keyword arguments (very readable)
prompt = build_prompt(my_note, style="json", max_words=300)

# Return multiple values (returns a tuple)
def count_tokens(text: str):
    words = len(text.split())
    chars = len(text)
    return words, chars          # returns tuple

w, c = count_tokens("Hello world")  # unpacking

You know Java OOP. Python OOP is similar but lighter. Here's a practical example — an Agent class you'll actually use:

PYTHON — Class: NoteAgent real project example 
class NoteAgent:
    """An agent that converts raw notes to structured documents."""

    # Class variable (shared by all instances)
    version = "1.0.0"

    def __init__(self, api_key: str, model: str = "gemini-1.5-flash"):
        # self = this in Java
        self.api_key = api_key
        self.model   = model
        self.token_count = 0
        self.history = []         # list of processed notes

    def process_note(self, raw_note: str) -> dict:
        """Send note to AI and return structured result."""
        cleaned = self._clean_input(raw_note)
        prompt  = self._build_prompt(cleaned)
        result  = self._call_api(prompt)
        self.history.append(result)
        return result

    def _clean_input(self, text: str) -> str:
        """Private method (convention: prefix with _)"""
        return text.strip()

    def _build_prompt(self, note: str) -> str:
        return f"Structure this note as JSON:\n\n{note}"

    def _call_api(self, prompt: str) -> dict:
        # Will be implemented in the API section
        pass

    def get_stats(self) -> dict:
        return {"notes_processed": len(self.history),
                "tokens_used": self.token_count}

# Usage
agent = NoteAgent(api_key="abc123")
result = agent.process_note("meeting jan 15 discuss roadmap q1 ...")
print(agent.get_stats())
💡
In Python, there are no true private attributes like in Java. By convention, prefix with _method to signal "internal use only". The class still works the same.

Error Handling

API calls will fail sometimes — network issues, rate limits, bad responses. Good error handling is what separates a prototype from a reliable agent.

PYTHON — Retry Logic with try/except like try/catch in Java 
import time

def call_api_safely(prompt: str, retries: int = 3) -> str:
    """Call API with retry logic — essential for production."""

    for attempt in range(retries):        # range(3) → loops 0, 1, 2
        try:
            response = api.generate(prompt)
            return response.text           # success → exit the function immediately

        except ConnectionError as e:      # network down / timeout
            print(f"Network error (attempt {attempt+1}): {e}")
            time.sleep(2)                # wait 2s, then loop continues → retry

        except ValueError as e:           # API replied but sent garbage JSON
            print(f"Bad response format: {e}")
            return None                  # retrying is pointless → stop now

        except Exception as e:            # anything unexpected
            print(f"Unexpected error: {e}")
            raise                         # re-throw → like "throw e" in Java

    # This line is only reached if ALL 3 attempts failed with ConnectionError
    print("All retries failed.")
    return None
⚙️ How the retry loop works

for attempt in range(3) runs the block up to 3 times (attempt = 0, 1, 2). If return response.text is hit on attempt 0, the function exits immediately — no more iterations. If a ConnectionError happens instead, time.sleep(2) pauses execution for 2 seconds, then the loop naturally continues to attempt 1. The retry only happens for ConnectionError — not for the other two except blocks, which both exit the function early.

⚖️ Three different strategies

ConnectionError → wait and retry (network might come back). ValueError → return None and stop (retrying gives the same garbage). Exception → raise with no argument re-throws the current exception up the call stack — exactly like throw e in Java. Never silently swallow unknown errors: you need to know they happened.

PYTHON — Custom Exception + Length Check like "extends Exception" in Java 
# Define a custom exception — like "public class TokenLimitError extends Exception" in Java
class TokenLimitError(Exception):
    """Raised when a note exceeds the token limit."""
    pass                               # "pass" = empty body (parent handles everything)

def check_note_length(note: str, limit: int = 4000):
    # note.split()  → splits by spaces → gives a LIST of words
    # len(...)      → counts items in the list = word count
    if len(note.split()) > limit:
        raise TokenLimitError(f"Note too long: {len(note.split())} words")
        # ↑ like "throw new TokenLimitError(...)" in Java

# How to use it:
try:
    check_note_length(my_note)
    result = call_api_safely(my_note)
except TokenLimitError as e:
    print(f"Note rejected: {e}")       # handle gracefully
🧩 What is pass?

Python requires at least one statement inside every class or function body. pass is a no-op placeholder that means "nothing to add here". Since TokenLimitError inherits everything from Exception (the message, the stack trace, etc.), there is genuinely nothing to add — so pass is correct and complete.

🔍 Why .split() for word count?

"hello world foo".split() returns ["hello", "world", "foo"] — a list of words split by whitespace. len() then counts the list items. This is a fast, good-enough approximation of token count. Real tokenizers are more complex, but for routing decisions (short vs long note), word count works perfectly.

Calling AI APIs

Here's how to call Gemini and Groq from Python. Both follow the same pattern: send a prompt → receive structured text.

PYTHON — Google Gemini API pip install google-generativeai 
import google.generativeai as genai   # "as genai" = nickname, like an alias
import os, json                         # two imports on one line

# Authenticate once at the top — reads your key from .env
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

def structure_note_gemini(raw_note: str) -> dict:
    # Instantiate the model — like: GenerativeModel model = new GenerativeModel("...") in Java
    model = genai.GenerativeModel("gemini-1.5-flash")

    # f"""...""" = multi-line f-string. {raw_note} is injected at the bottom.
    # The instructions above it guide the AI — the wording matters a lot.
    prompt = f"""
Convert the following messy note into a structured JSON document.
Return ONLY valid JSON with these fields:
- title (string)
- date (string or null)
- participants (list of strings)
- summary (string, 2-3 sentences)
- action_items (list of strings)
- tags (list of keywords)

Note:
{raw_note}
    """

    response = model.generate_content(prompt)
    text = response.text.strip()         # remove surrounding whitespace/newlines

    # The AI sometimes wraps its answer in markdown backticks like:
    # ```json
    # { "title": "Meeting" }
    # ```
    # We need to strip that wrapper before parsing.
    if text.startswith("```"):
        text = text.split("```")[1]      # split by ``` → take the middle part
        if text.startswith("json"):
            text = text[4:]               # skip the word "json" (4 characters)

    return json.loads(text)              # JSON string → Python dict (like JSON.parse())

# --- Test it ---
raw = "jan 15 meet w alice bob q1 goals fix the auth bug deploy next week"
result = structure_note_gemini(raw)
print(json.dumps(result, indent=2))   # dict → formatted JSON string (opposite of loads)
🔀 json.loads vs json.dumps

loads (load from string) converts a JSON string into a Python dict — what you use when the API sends you data. dumps (dump to string) does the opposite — converts a dict back into a formatted JSON string — what you use to print or save. The s stands for "string" in both cases.

🧹 Why strip the backticks?

Even when you say "Return ONLY valid JSON", the AI sometimes adds a markdown code fence around its answer (```json ... ```). text.split("```")[1] splits the string by the backtick fence and takes index [1] — the middle piece — which is the raw JSON. text[4:] then skips the 4-character word "json" that follows the opening fence.

PYTHON — Groq API (ultra-fast) pip install groq 
from groq import Groq
import os, json

# Create the client once — holds your credentials
client = Groq(api_key=os.getenv("GROQ_API_KEY"))

def structure_note_groq(raw_note: str) -> dict:
    completion = client.chat.completions.create(
        model="llama3-8b-8192",        # model name: llama3, 8 billion params, 8192 token ctx

        # Groq uses a chat-style API: you send a LIST of messages with roles.
        # "system" = background instructions (like a job description for the AI)
        # "user"   = the actual request (like what you'd type in a chat)
        messages=[
            {
                "role": "system",
                "content": "You are a note structuring assistant. Always respond with valid JSON only."
            },
            {
                "role": "user",
                "content": f"Structure this note as JSON with: title, date, summary, action_items, tags.\n\n{raw_note}"
            }
        ],

        temperature=0.2,   # 0.0 = robotic/deterministic, 1.0 = creative/random. Low = consistent JSON
        max_tokens=1024,  # max length of the AI's response
    )

    # Navigate the response object to get the text:
    # completion.choices  → list of possible responses (usually just 1)
    # [0]                 → take the first one
    # .message.content    → the actual text string
    text = completion.choices[0].message.content
    return json.loads(text)             # JSON string → Python dict
💬 The system / user roles

The messages list simulates a conversation. system sets the AI's global behavior — like giving an employee their job description before they start working. user is the specific task for this call. You send both together every time. A third role, assistant, would hold the AI's previous replies if you were building a multi-turn conversation — not needed here.

🌡️ What is temperature?

Temperature controls how "creative" (random) the AI's output is. At 0.0 it always picks the most probable next word — fully deterministic. At 1.0 it's much more varied. For JSON output you want 0.1–0.3: low enough to produce consistent, parseable structure, but not so low that it gets stuck in repetition loops.

PYTHON — Smart Router (split work between APIs) token economy strategy 
# Route each note to the right API based on its length
def smart_route(raw_note: str) -> dict:
    word_count = len(raw_note.split())   # split by spaces → list of words → count them

    if word_count < 100:
        print(f"→ Routing to Groq ({word_count} words)")
        return structure_note_groq(raw_note)    # fast, free, small context
    else:
        print(f"→ Routing to Gemini ({word_count} words)")
        return structure_note_gemini(raw_note)  # 1M token context window

# ── Process every .txt file in the notes/ folder ──────────────────────
import os

for filename in os.listdir("notes/"):       # like File[].listFiles() in Java
    if filename.endswith(".txt"):
        # "with open(...) as f:"  →  like try-with-resources in Java
        # The file closes automatically when the indented block ends.
        with open(f"notes/{filename}") as f:
            note = f.read()                 # reads entire file as one string

        result = smart_route(note)           # dict returned by whichever API was chosen

        # json.dump  → write dict to FILE  (no 's')
        # json.dumps → write dict to STRING (with 's')
        with open(f"output/{filename}.json", "w") as out:
            json.dump(result, out, indent=2)
🗂️ with open(...) as f: explained

This is Python's equivalent of Java's try-with-resources. It opens the file, gives you a handle f, and automatically closes the file when the indented block exits — even if an exception occurs. Without it, you'd need to manually call f.close(). Always use with open for file I/O.

🔌 The Strategy Pattern in 4 lines

smart_route() is the Strategy Pattern — same interface, different implementations chosen at runtime based on a condition (word count). In Java you'd likely define an interface and two implementing classes. In Python, a simple if/else calling two different functions achieves the same result with far less boilerplate.

PROJECT STRUCTURE recommended layout 
my-agent/
├── .env                    # API keys — NEVER commit to git
├── .gitignore              # tells git to ignore .env, venv/, __pycache__/
├── requirements.txt        # like pom.xml: list of all pip dependencies
├── README.md
│
├── notes/                  # INPUT: drop your raw .txt files here
│   ├── meeting_jan15.txt
│   └── ideas_q1.txt
│
├── output/                 # OUTPUT: structured .json and .md files land here
│   ├── meeting_jan15.json
│   └── meeting_jan15.md
│
├── src/                    # all Python source code lives here
│   ├── config.py           # loads .env → exposes GEMINI_KEY, GROQ_KEY
│   ├── agent.py            # NoteAgent class — the main brain
│   ├── router.py           # smart_route() function
│   ├── apis/               # one file per API — easy to add more
│   │   ├── gemini.py
│   │   └── groq.py
│   └── processors/
│       ├── cleaner.py      # text cleaning utilities
│       └── formatter.py    # converts dict → Markdown or JSON string
│
└── main.py                 # entry point — like public static void main()
📦 Why src/ subfolder?

Putting all code inside src/ separates your source from config files, input data, and output. When you import between files you write from src.apis.gemini import structure_note_gemini — Python reads the dots as folder separators. Each folder with Python files needs an empty __init__.py file so Python treats it as a package (module group).

🙈 What goes in .gitignore?

At minimum: .env (secrets), venv/ (the virtual environment — hundreds of MB, never committed), __pycache__/ (Python's compiled bytecode cache), and output/ (generated files). Your teammates regenerate these locally. Only source code and config templates go into git.

PYTHON — main.py entry point ties everything together 
# "from src.agent import NoteAgent"
# → go into the src/ folder, open agent.py, import the NoteAgent class
# → like "import src.agent.NoteAgent" in Java
from src.agent import NoteAgent
from src.config import GEMINI_KEY, GROQ_KEY
import os, json

def main():
    agent = NoteAgent(gemini_key=GEMINI_KEY, groq_key=GROQ_KEY)

    # List comprehension: build a list of .txt filenames in one line
    # Equivalent Java: list.stream().filter(f -> f.endsWith(".txt")).collect(...)
    note_files = [f for f in os.listdir("notes") if f.endswith(".txt")]

    print(f"Found {len(note_files)} notes to process...")

    for filename in note_files:
        with open(f"notes/{filename}", encoding="utf-8") as f:
            raw = f.read()                # entire file as one string

        print(f"Processing: {filename}")
        result = agent.process_note(raw)  # returns a dict

        # filename.replace('.txt', '.json') → "meeting_jan15.txt" → "meeting_jan15.json"
        out_path = f"output/{filename.replace('.txt', '.json')}"
        with open(out_path, "w") as f:
            # ensure_ascii=False → allows Arabic, French, accented chars to save as-is
            # without it, "é" would be saved as "\u00e9"
            json.dump(result, f, indent=2, ensure_ascii=False)

    print(f"\n✅ Done! Stats: {agent.get_stats()}")

# This block runs ONLY when you execute "python main.py" directly.
# If another file imports main.py, this block is skipped.
# Equivalent to: public static void main(String[] args) in Java.
if __name__ == "__main__":
    main()
❓ What is if __name__ == "__main__"?

Every Python file has a built-in variable __name__. When you run a file directly (python main.py), Python sets __name__ to "__main__". When another file imports it, __name__ becomes the module name instead. This guard ensures your startup code only runs when you intend it to — exactly like Java's public static void main() entry point.

🌍 Why ensure_ascii=False?

By default, json.dump escapes every non-ASCII character: the French "é" becomes the ugly \u00e9 in your output file. Setting ensure_ascii=False tells Python to write characters as-is in UTF-8. Since your notes might contain Arabic, French, or accented characters, always set this to False.

Frameworks — Phase 2

Once your raw Python agent is working, these frameworks will supercharge it:

🦜

LangChain

The most popular agent framework. Chains, memory, tool use, retrievers. Great docs and community. Start here.

🌊

LangGraph

Build stateful agents as graphs (nodes + edges). Perfect when your agent has complex multi-step decision logic.

🦙

LlamaIndex

Specialized in document ingestion, indexing, and RAG. Ideal if you want your agent to search over many notes.

🤝

CrewAI

Orchestrate multiple specialized agents as a "crew". Good for splitting work: one agent cleans, one structures, one formats.

⚠️
Don't jump to frameworks too early. Build your agent in raw Python first. This forces you to understand what the frameworks are actually doing under the hood — which makes debugging much easier later.
